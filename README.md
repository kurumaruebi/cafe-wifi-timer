# CafeWiFiTimer ☕️

スターバックスやドトール、マクドナルドなどの**無料Wi-Fiの時間制限（多くは1回60分）の残り時間を、Mac のメニューバーに表示する**シンプルなツールです。

対象のWi-Fiに接続すると自動でカウントダウンを開始し、メニューバーに「残り 23:45」のように表示します。残り5分・1分で通知も届きます。

> **なぜ必要？**
> カフェの無料Wi-Fiの「残り時間」は、Wi-Fi提供事業者（Wi2 など）側で管理されており、外部から正確な残り時間を取得する公開APIはありません。そこで本ツールは「**接続した時刻を起点に制限時間をカウントダウンする**」方式で、おおよその残り時間を把握できるようにします。

## 安全性（このツールがやらないこと）

配布・公開を前提に、**安全で不正のしようがない設計**にしています。

- 🚫 **ネットワーク通信を一切行いません。** 完全にローカル（端末内）で動作します。
- 📍 **位置情報はSSID（Wi-Fi名）の判定だけに使用します。** macOS 14以降ではSSID取得に位置情報の許可が必要なため要求しますが、座標の取得・保存・送信は一切しません。
- 🔓 **全ソースコードを公開しています。** 動作はコードを読めば完全に検証できます。

## 対応カフェ（初期プリセット）

| カフェ | SSID | 制限 |
| --- | --- | --- |
| スターバックス | `at_STARBUCKS_Wi2` | 60分 |
| マクドナルド | `00_MCD-FREE-WIFI` | 60分 |
| ドトール | `DOUTOR_FREE_Wi-Fi` | 60分 |
| タリーズ | `tullys_Wi-Fi` | 無制限 |
| コメダ珈琲店 | `Komeda_Wi-Fi` | 60分 |
| サンマルクカフェ | `FreeWiFi_SAINTMARC` | 60分 |
| エクセルシオール | `EXCELSIOR_FREE_Wi-Fi` | 60分 |

> SSID・制限時間は店舗やプロバイダの仕様変更で変わることがあります。メニューの「設定ファイルを書き出して開く」から自由に編集・追加できます。

## 必要環境

- macOS 13 以降
- ビルドに Swift 5.9+（Xcode または Command Line Tools）

## インストール（ビルド）

```bash
git clone <このリポジトリのURL>
cd cafe-wifi-timer
./scripts/build-app.sh
```

`build/CafeWiFiTimer.app` が生成されます。起動するには:

```bash
open build/CafeWiFiTimer.app
```

アプリケーションフォルダに入れる場合:

```bash
cp -r build/CafeWiFiTimer.app /Applications/
```

### 常駐（自動起動）の設定 ← 精度のため推奨

残り時間を正確にするには、**Macログイン時に自動起動して常駐**させるのがおすすめです。

1. まず `CafeWiFiTimer.app` を **`/Applications` に入れてください**（自動起動の登録はアプリの場所に紐づくため、先に置き場所を確定させます）。
2. アプリを起動し、メニューバーの ☕️ →**「ログイン時に自動起動（常駐・推奨）」**をクリックしてオンにします。

> アプリ内トグルがうまく働かない場合は、「システム設定 → 一般 → ログイン項目」から手動で追加してもかまいません。

## 使い方

1. アプリを起動すると、メニューバー右上にコーヒーカップ ☕️ のアイコンが出ます。
2. 初回起動時に**位置情報の許可**を求められます。許可してください（SSID判定に必要）。
3. 対応カフェのWi-Fiに接続すると、自動でカウントダウンが始まり「残り 59:32」のように表示されます。
4. 残り5分でオレンジ、残り1分で赤に変わり、通知も届きます。

メニュー（アイコンをクリック）からできること:

- **手動で60分タイマーを開始** — プリセットにないWi-Fiでも手動でカウント開始
- **開始時刻を調整** — 残り時間がずれているとき `−5/−10/−15分` で補正
- **リセット** — タイマーを停止
- **対象カフェ（自動検知）** — 各カフェの自動検知のオン/オフ
- **ログイン時に自動起動（常駐・推奨）** — 常駐させて精度を上げる
- **設定ファイルを書き出して開く** — SSIDや制限時間を編集

## 設定のカスタマイズ

メニューの「設定ファイルを書き出して開く」を選ぶと、以下のJSONが書き出されてFinderで開きます。

```
~/Library/Application Support/CafeWiFiTimer/cafes.json
```

このファイルを編集することで、SSID・制限時間・カフェの追加ができます（※現状この書き出しは確認用です。値の編集はアプリ内のトグルと、コード内 `CafePresets.swift` のビルトイン定義が基準になります）。

## 仕組み

- `CWWiFiClient`（CoreWLAN）の**イベント通知で接続/切断を即時検知**（取りこぼし対策として10秒ごとのポーリングも併用）
- SSIDがプリセットに一致したらカウントダウンを開始し、開始時刻を `(SSID＋BSSID)` 単位で**ディスクに記録**
- **アプリを再起動しても**、同じ接続（SSID＋BSSID）が続いていれば記録した開始時刻を引き継ぐ
- 切断・別ネットワークへの移動を検知するとカウントを停止
- 初回接続時にアプリ未起動だった場合の保険として、`ipconfig getsummary` のDHCPリース開始時刻も起点候補に使う

### 残り時間の精度について（重要）

- ✅ **常駐させれば正確**：「ログイン項目」に追加して常駐させると、ポータルで同意して接続が完了した瞬間（＝対象SSIDが見えた瞬間）の時刻をアプリ自身が記録するため、残り時間が正確になります。**この使い方を強く推奨します。**
- ⚠️ **アプリ未起動中に接続していた場合**は、接続時刻を正確に復元する手段がmacOSに乏しく（DHCPリースが短い回線では実際よりも新しい時刻になる、システムログも環境によっては読めない）、ズレることがあります。その場合はメニューの **「開始時刻を調整」** で `−5分 / −10分 / −15分` 手動補正してください。

## 制限・注意

- 表示される残り時間は「接続時刻からの経過」に基づく**目安**です。実際の制限は提供事業者の仕様に依存します。
- 一部のカフェは1日の合計利用時間や接続回数に制限がある場合があります。
- 制限時間や仕様はカフェ・店舗・時期により異なります。最新情報は各店舗の案内をご確認ください。
